ACTIVIDAD 8: Sistema de Inicio de Sesión con Arduino

Descripción del Proyecto

Crear un sistema de inicio de sesión donde:

  • Si la contraseña es correcta: mostrar "CORRECTO" en el LCD y encender un LED verde
  • Si la contraseña es incorrecta: activar una alarma (buzzer) y mostrar "INCORRECTO" en el LCD

1. OBJETIVOS

1.1 Objetivo General

Desarrollar e implementar un sistema de control de acceso basado en Arduino que valide contraseñas mediante un teclado matricial, proporcionando retroalimentación visual y auditiva según la autenticación del usuario.

1.2 Objetivos Específicos

  1. Programar un microcontrolador Arduino para el procesamiento y validación de contraseñas ingresadas por teclado matricial.
  2. Implementar un sistema de retroalimentación visual mediante pantalla LCD 16x2 para mostrar mensajes de estado.
  3. Configurar indicadores LED para señalización visual del estado de autenticación (acceso autorizado/denegado).
  4. Integrar un sistema de alarma sonora mediante buzzer para alertas de seguridad en caso de acceso no autorizado.
  5. Simular y validar el funcionamiento del sistema utilizando la plataforma Tinkercad para verificar la lógica de programación.

2. MARCO TEÓRICO

2.1 Microcontrolador Arduino

Arduino es una plataforma de desarrollo de hardware y software libre basada en microcontroladores. Utiliza un lenguaje de programación similar a C++ y permite la creación de proyectos interactivos mediante la conexión de sensores, actuadores y otros componentes electrónicos.

2.2 Componentes del Sistema

Teclado Matricial 4x4

Dispositivo de entrada que permite al usuario ingresar datos numéricos y alfabéticos. Funciona mediante un sistema de filas y columnas que se activan al presionar las teclas.

Pantalla LCD 16x2

Display de cristal líquido capaz de mostrar 32 caracteres distribuidos en 2 filas de 16 caracteres cada una. Utiliza el protocolo I2C para comunicación con el Arduino.

LED (Diodo Emisor de Luz)

Componente semiconductor que emite luz cuando circula corriente eléctrica a través de él. Se utiliza como indicador visual del estado del sistema.

Buzzer Piezoeléctrico

Dispositivo electroacústico que genera sonido mediante la vibración de un material piezoeléctrico. Se emplea como sistema de alarma audible.

2.3 Sistemas de Control de Acceso

Los sistemas de control de acceso son mecanismos de seguridad que regulan quién puede acceder a recursos específicos. En el contexto de sistemas embebidos, estos sistemas validan credenciales (como contraseñas) y ejecutan acciones basadas en el resultado de la autenticación.

Principios de Funcionamiento:

  • Autenticación: Verificación de la identidad del usuario
  • Autorización: Determinación de permisos de acceso
  • Auditoría: Registro de intentos de acceso
  • Retroalimentación: Comunicación del resultado al usuario

3. SIMULACIÓN EN TINKERCAD

3.1 ¿Qué es Tinkercad?

Tinkercad es una plataforma web gratuita desarrollada por Autodesk que permite el diseño, simulación y programación de circuitos electrónicos de forma virtual. Incluye una amplia biblioteca de componentes electrónicos y permite la programación de microcontroladores mediante bloques visuales o código tradicional.

3.2 Ventajas de la Simulación

  • Prototipado rápido: Permite probar diseños sin componentes físicos
  • Depuración eficiente: Facilita la identificación y corrección de errores
  • Experimentación segura: Evita daños a componentes reales por conexiones incorrectas
  • Accesibilidad: No requiere hardware físico para el aprendizaje
  • Visualización clara: Muestra el comportamiento del circuito en tiempo real

4. FUNCIONAMIENTO DEL SISTEMA

4.1 Algoritmo de Funcionamiento

  1. Inicialización: El sistema se inicia mostrando un mensaje de bienvenida en el LCD
  2. Entrada de datos: El usuario ingresa la contraseña usando el teclado matricial
  3. Procesamiento: Arduino compara la contraseña ingresada con la almacenada
  4. Validación positiva: Si coinciden, se muestra "CORRECTO" y se enciende el LED verde
  5. Validación negativa: Si no coinciden, se muestra "INCORRECTO" y se activa la alarma
  6. Reset: Después de un tiempo determinado, el sistema vuelve al estado inicial

4.2 Estados del Sistema

Estado de Espera: LCD muestra "Ingrese Password", sistema listo para recibir entrada

Estado de Ingreso: Usuario escribiendo contraseña, caracteres mostrados como asteriscos

Estado de Validación: Sistema procesando y comparando contraseña

Estado de Éxito: Acceso autorizado, LED verde encendido, mensaje "CORRECTO"

Estado de Error: Acceso denegado, alarma sonando, mensaje "INCORRECTO"

5. CONCLUSIONES

5.1 Logros Alcanzados

  • Se implementó exitosamente un sistema de control de acceso funcional utilizando componentes básicos de Arduino.
  • La integración del teclado matricial 4x4 permitió una interfaz de usuario intuitiva para el ingreso de contraseñas.
  • El sistema de retroalimentación dual (visual y auditiva) proporciona una experiencia de usuario clara y efectiva.
  • La simulación en Tinkercad demostró ser una herramienta valiosa para el desarrollo y validación del proyecto antes de la implementación física.

5.2 Aprendizajes Técnicos

  • Programación de microcontroladores: Se adquirieron conocimientos sobre el manejo de entradas digitales, control de displays LCD y gestión de estados en Arduino.
  • Interfaz de usuario: Se comprendió la importancia de proporcionar retroalimentación clara y inmediata al usuario en sistemas interactivos.
  • Sistemas de seguridad: Se exploraron conceptos básicos de autenticación y control de acceso en sistemas embebidos.
  • Integración de componentes: Se desarrolló experiencia en la conexión y programación de múltiples componentes trabajando en conjunto.

5.3 Aplicaciones Potenciales

Este proyecto sienta las bases para sistemas más complejos como:

  • Cerraduras electrónicas para hogares y oficinas
  • Sistemas de control de acceso a laboratorios o áreas restringidas
  • Cajas fuertes digitales
  • Sistemas de seguridad vehicular
  • Control de acceso a equipos industriales

5.4 Mejoras Futuras

  • Múltiples usuarios: Implementar un sistema que soporte varias contraseñas válidas
  • Registro de intentos: Almacenar historial de accesos exitosos y fallidos
  • Temporización de bloqueo: Bloquear el sistema temporalmente tras varios intentos fallidos
  • Comunicación inalámbrica: Enviar notificaciones de acceso via WiFi o Bluetooth
  • Encriptación: Implementar algoritmos de cifrado para mayor seguridad

5.5 Reflexión Final

El desarrollo de este sistema de inicio de sesión con Arduino ha demostrado cómo los conceptos básicos de programación y electrónica pueden combinarse para crear soluciones prácticas de seguridad. La experiencia obtenida mediante la simulación en Tinkercad ha sido fundamental para comprender el comportamiento del sistema antes de su implementación física, destacando la importancia de las herramientas de simulación en el proceso de desarrollo de sistemas embebidos.
